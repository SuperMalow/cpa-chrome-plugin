# Contributing

感谢你愿意为这个项目贡献代码。

本文档主要说明如何在本地运行项目、提交修改，以及在提交前需要注意的事项。

## 开发环境

建议环境：

- `Node.js 18+`
- `npm 9+`
- 最新版 `Chrome`

## 本地启动

```bash
npm install
npm run dev
```

开发时通常会同时使用两种方式：

- `http://localhost:5173` 调试完整页面
- `npm run build` 后，将 `dist/` 作为未解压扩展加载到 Chrome 中调试 Popup

## 提交前检查

提交前请至少完成以下检查：

```bash
npm run build
```

如果你的改动涉及以下内容，也请一起自查：

- 设置页保存逻辑
- Popup 是否能正常读取配置
- 日间 / 夜间模式切换
- 账号启用 / 停用 / 删除后的刷新逻辑
- Chrome 扩展加载是否正常

## 代码风格

请尽量遵循当前仓库已有约定：

- 使用 `Vue 3 Composition API`
- 优先使用 `<script setup>`
- 样式优先使用 `Tailwind CSS`
- 页面拆分成可维护的业务组件，不把过多逻辑堆在一个文件里
- 与现有 UI 风格保持一致：
  - 轻玻璃感
  - 胶囊式按钮
  - 不使用 hover scale 放大
  - 兼容日夜间模式

## Pull Request 建议

建议每个 PR 聚焦单一主题，例如：

- 只修复一个接口问题
- 只优化一个页面
- 只新增一个能力

这样更方便 review，也更容易回归验证。

建议在 PR 描述里写清楚：

- 改动目标
- 主要改了哪些页面 / 组件 / 数据层
- 如何手动验证
- 是否涉及接口变更

## 不应提交的内容

请不要提交以下内容：

- 真实接口密钥
- 内部服务账号
- `.env` 中的敏感配置
- `dist.pem`
- 其他任何私钥、签名文件或个人隐私数据

说明：

- `dist.crx` 是可分发的扩展安装包
- `dist.pem` 是扩展签名私钥，必须严格保密

## Issue 建议

提交 Issue 时，建议尽量提供：

- 浏览器版本
- 插件版本
- 问题出现页面
- 复现步骤
- 控制台报错
- 是否与 `CORS`、接口鉴权、扩展重载有关

这样会更容易快速定位问题。
