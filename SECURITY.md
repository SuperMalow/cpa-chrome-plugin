# Security Policy

## 支持范围

当前仓库主要用于 Chrome 插件形态的 CPA 面板聚合与管理。

如果你发现以下类别的问题，建议优先按安全问题处理：

- 密钥泄露
- 扩展私钥泄露
- 配置被非预期读取
- 可绕过鉴权调用敏感接口
- 可在插件内注入恶意脚本
- 会导致敏感服务地址、账号信息泄露的问题

## 敏感信息说明

仓库运行过程中可能涉及以下敏感数据：

- CPA API Key / Bearer Token
- 账号文件名
- 服务地址
- 注册机面板地址
- Chrome 扩展签名私钥 `dist.pem`

请注意：

- 不要把真实密钥写进代码
- 不要在公开 Issue 中粘贴真实请求头
- 不要提交 `dist.pem`
- 不要把包含敏感信息的截图直接公开

## 漏洞反馈建议

如果问题涉及真实密钥、真实账号、越权访问或利用细节，**不要直接公开提交完整复现细节**。

建议先联系仓库维护者，确认修复窗口后再决定是否公开披露。

如果暂时只能通过公开 Issue 反馈，请至少做以下处理：

- 打码密钥
- 打码地址中的敏感路径
- 去掉账号隐私信息
- 不公开可直接利用的攻击步骤

## 本仓库当前的安全注意事项

### 1. CORS 与浏览器请求限制

当前插件会直接从浏览器访问 CPA 接口。

这意味着后端需要正确处理：

- `Origin`
- `Authorization`
- `OPTIONS` 预检

如果后端 CORS 配置不当，请求可能失败；如果配置过宽，也可能带来额外风险。

### 2. 本地存储

设置优先保存在 `chrome.storage.local`，在非扩展环境下回退到 `localStorage`。

因此：

- 请避免在不可信环境中长期保存真实生产密钥
- 调试完成后建议清理无用配置

### 3. 扩展签名文件

如果你通过浏览器打包扩展，会得到：

- `dist.crx`
- `dist.pem`

其中：

- `dist.crx` 可用于安装或分发
- `dist.pem` 是私钥，必须严格保密

一旦泄露，其他人可能以相同扩展身份重新打包扩展。
