# CPA 监控面板 Chrome 插件

这是一个用于查看 CPA 服务状态、账号状态和额度信息的 Chrome 插件。

你只需要把项目构建成 `dist/`，然后在 Chrome 扩展管理页加载这个 `dist/` 文件夹即可使用。

## 快速使用

### 1. 安装依赖

```bash
npm install
```

### 2. 构建插件

```bash
npm run build
```

构建完成后会生成 `dist/` 文件夹。

### 3. 在 Chrome 中加载 `dist/`

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目生成的 `dist/` 文件夹
5. 点击浏览器右上角扩展图标，打开 CPA 快捷面板

> 注意：不要直接把 `dist/` 拖进浏览器窗口。正确方式是通过 `chrome://extensions/` 加载已解压扩展。

## 第一次配置

1. 打开插件后，进入“设置”
2. 新增一个 CPA 接入
3. 填写 CPA 地址、密钥、鉴权方式和超时时间
4. 点击测试连接
5. 保存配置

配置保存后，可以在：

- 快捷面板中查看服务摘要
- 运营面板中查看请求、Token、健康状态
- 账号页中查看账号列表、启停账号、刷新额度

## 常用功能

- **快捷面板**：查看成功率、当前小时请求、Token 消耗、问题账号数
- **运营面板**：查看请求趋势、Token 趋势、服务健康状态
- **账号管理**：查看认证账号，支持搜索、筛选、分页
- **账号操作**：启用、停用、删除账号
- **额度刷新**：
  - `远端本页刷新`：本页账号通过 CPA 后端代理刷新额度
  - `本地本页刷新`：本页账号在浏览器本机直接请求 ChatGPT usage 接口刷新额度
  - 表格内的 `远端` / `本地`：只刷新当前这一行账号
- **批量处理**：停用超额账号、恢复停用超过 7 天的账号
- **主题切换**：支持日间 / 夜间模式

## 页面入口

- 插件图标：打开 Popup 快捷面板
- 完整面板：查看运营数据
- 账号页：管理认证账号和刷新额度
- 设置页：维护 CPA 接入配置和常用链接

## 配置保存在哪里？

插件优先使用 `chrome.storage.local` 保存配置。

保存内容包括：

- CPA 接入列表
- 每个接入的名称、地址、密钥、鉴权方式、超时时间、启用状态
- 注册机面板链接
- 主题模式

如果不是在 Chrome 扩展环境运行，则回退到浏览器 `localStorage`。

## 常见问题

### 为什么接口请求失败并提示 CORS？

因为插件会从浏览器直接请求 CPA 服务。CPA 后端需要允许跨域请求，并放行：

- `Origin`
- `Authorization`
- `OPTIONS` 预检请求

### 为什么保存设置后重新打开扩展看不到配置？

先进入设置页重新保存一次配置，然后刷新扩展重试。

### 为什么必须加载 `dist/` 文件夹？

`dist/` 是构建后的未解压扩展目录。Chrome 扩展管理页只能加载这种目录作为开发模式扩展。
